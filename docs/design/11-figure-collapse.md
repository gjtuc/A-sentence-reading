# 11 — Figure collapse (드래그 스플리터)

## 기본 분할

기본 상태(`is-default-split`): 문장 패널은 **내용 높이만**(스크롤 없음), 그림 패널이 남은 세로를 가져간다.  
스플리터 드래그·그림 확대 시 수동 높이 모드로 전환.

## 문제

그림이 세로를 많이 쓰면 문장 읽는 영역이 줄어든다. 그림을 잠시 접고 **문장 패널을 키운다**.

## 동작

1. 문장/그림 사이 **점선 핸들**을 **아래로** 드래그 → 아래 그림 높이가 줄며 접힘.
2. 임계값 이하 → **스냅**: 하단에 `그림 프레임` 얇은 바만 남김.
3. 접힌 `그림 프레임`을 **위로** 드래그(또는 클릭) → 이전 펼침 높이로 복원.
4. 문장 패널은 `flex: 1`로 남는 세로 공간을 차지.
5. **문장 박스 클릭** 또는 **`↑`** → 그림 접힘(문장 영역 최대). 프레임 테두리 숨김(배경과 동일). **다시 클릭/`↑`** → 복원.
5b. **브라우저 전체화면(`F`) + 문장 확대** → 글만 남김. 문장 패널 hover 시에만 크롬(라벨·화살표·힌트·그림 프레임) 표시.
5c. **브라우저 전체화면에서 `↓`/`↑`** → 중간 large 없이 그림 몰입 ↔ 글 몰입을 부드럽게 전환.
6. **그림 박스 클릭 (1회)** 또는 **`↓`** → 그림 높이 ≈ 세로의 78%.
7. **그림 박스 클릭 (2회)** 또는 **`↓` 다시** → **전체화면**.
8. 전체화면에서 다시 클릭/`↓` 또는 `Esc` → large로 복귀.

## 비동작

- 접어도 **그림·문장 인덱스는 그대로**.
- 접힌 동안 Shift+←/→ 로 그림 인덱스는 변경 가능 (바의 Fig i/N). 미리보기는 펼쳐야 보임.

## 수치 (1차)

| 토큰 | 값 |
|------|-----|
| `--figure-collapsed` | `48px` |
| snap 접기 임계 | 높이 &lt; `110px` → collapsed |
| 기본 펼침 높이 | `340px` |
| transition | 기본 0.4s/0.75s. **브라우저 FS ↑↓만** 아래↔위 슬라이드 2.25s |
| 드래그 식 | `height = startH - dy` (아래 그림 기준) |

## 상태

```
figureLayout: "expanded" | "collapsed"
figureHeightPx: number
```

`localStorage` 키: `asr.figureLayout.v1`

## 접근성

- `role="separator"` · 더블클릭 토글
- 포커스 후 `ArrowDown` 접기, `ArrowUp` 펼치기/키우기, `Enter` 토글

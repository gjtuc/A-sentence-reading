# UX

페인트 목업과 Immersive Reader 감각을 코드로 옮기기 위한 화면 계약.

## 레이아웃

```
┌──────────────────────────────────────────┐
│  한 문장                 Sent j / M       │
│  [◀]     ┌──────────────┐      [▶]       │
│          │  one sentence │                │
│          │  (narrow col) │                │
│          └──────────────┘                │
├─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─┤
│  그림                    Fig i / N        │
│  [◀]     ┌──────────────┐      [▶]       │
│          │   figure     │                │
│          └──────────────┘                │
└──────────────────────────────────────────┘
```

- **상단**: 문장 하나. Immersive Reader식 타이포 (읽기 위치).
- **하단**: 그림 캐러셀. 캡션/번호는 가능하면 표시 (다음 단계).
- **구분선(스플리터)**: 아래로 드래그하면 그림이 접혀 **하단 그림 프레임**만 남고 문장 영역이 커진다. 프레임을 위로 끌면 복원. → [design/11-figure-collapse.md](design/11-figure-collapse.md)

## Immersive Reader 레시피 (문장 패널)

Azure SDK를 쓰지 않고 **CSS로 재현**한다. ([RESEARCH.md](RESEARCH.md) 참고)

| 설정 | CSS 방향 |
|------|----------|
| 어두운 배경 + 밝은 글자 | `--bg: #0f0f0f`, `--fg: #e3e3e3` |
| 좁은 열 | `max-width: ~27ch` (기존 대비 약 4/5) |
| 넓은 간격 | `letter-spacing`, `word-spacing`, `line-height` 증가 |
| 큰 글자 | `font-size` 1.25–1.75rem 대 |

선택 폰트: 시스템 가독 서체 또는 Sitka/Calibri 계열 폴백. Comic Sans는 Immersive Reader 옵션이지만 기본은 쓰지 않는다.

## 네비게이션

| 입력 | 동작 |
|------|------|
| `→` / 하단 ▶ | 다음 문장 |
| `←` / 하단 ◀ | 이전 문장 |
| `Shift+→` / 하단 ▶ | 다음 그림 |
| `Shift+←` / 하단 ◀ | 이전 그림 |
| (선택) `Home` / `End` | 문장 처음/끝 |
| `F` | 브라우저 전체화면 토글 (주소창·탭 숨김). 다시 `F` 또는 `Esc`로 종료 |
| (F 모드) `↑` | 바로 글 몰입(전체화면 크기 문장), 부드럽게 |
| (F 모드) `↓` | 바로 그림 몰입(전체화면 크기 그림), 부드럽게 · large 단계 없음 |
| (F + 문장 확대) | 글만 표시. 문장 영역에 마우스를 올리면 라벨·화살표·힌트·그림 프레임 일시 표시 |
| `↓` | 그림 프레임 클릭과 동일 (확대 → 전체화면 → 복귀) |
| 문장 박스 클릭 | 확대(그림 접기) · 다시 클릭 시 원래 크기 |
| 그림 박스 클릭 | 1회≈78% 확대 · 2회 전체화면 · 3회/Esc 복귀 |
| 그림 프레임 드래그 ↑ / 클릭 / 스플리터 더블클릭 | 그림 다시 펼침 |
| 스플리터 포커스 + ↓/↑ | 그림 접기/키우기, Enter 토글 |

**INVARIANT:** Shift 없는 화살표는 문장만, Shift는 그림만. 서로 묶지 않는다.  
**INVARIANT:** 그림 접기/펼치기는 **레이아웃만** 바꾸고 인덱스는 건드리지 않는다.

## 상태 표시

- `Fig 2 / 7`, `Sent 14 / 312` 처럼 **현재 / 전체**를 약하게 표시.
- 빈 목록이면 안내 문구 (“PDF를 넣으세요” — 다음 단계; 스켈레톤은 mock).

## 접근성 (초안)

- 버튼에 `aria-label`
- 포커스가 문장 패널에 있을 때 키보드 동작이 우선
- 색만으로 정보를 전달하지 않음 (인덱스 숫자 병기)

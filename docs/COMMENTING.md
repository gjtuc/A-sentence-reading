# Commenting rules

스켈레톤부터 주석을 **설계 문서의 연장**으로 쓴다. 코드가 자명하면 주석을 달지 않는다.

## 태그

| 태그 | 언제 |
|------|------|
| `# WHY:` | 왜 이 결정/경계인가 (제품·불변조건과 연결) |
| `# INVARIANT:` | 깨지면 제품 가설이 깨지는 조건 |
| `# NEXT:` | 스켈레톤 이후 구현할 것 (무엇을/대략 어떻게) |
| `# NOTE:` | 함정·논문 PDF 특수성 (약어, 다단, 스캔본) |

모듈 상단 docstring 형식:

```text
무엇을: ...
왜: ...
다음에: ...
```

## 언어

- `docs/`, README 제품 설명, HTML 주석: **한국어**
- 공개 식별자·API 경로: **영어**
- `# WHY:` 등 코드 주석: **한국어 가능** (유지보수 우선)

## 하지 말 것

- `# increment i by 1` 수준의 서술
- 주석으로 죽은 코드 보관
- `preferences` 문자열을 Immersive SDK처럼 손으로 조작하는 식의 미스터리 매직 (우리는 SDK 미사용)

## 리뷰 체크

파일을 열었을 때 **그림·문장 인덱스 독립**이 주석만으로도 보이는가?  
stub라면 **왜 NotImplemented인지**가 `# WHY:` / `# NEXT:`에 있는가?
